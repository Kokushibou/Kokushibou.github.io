reflection about basic language of Python

发表于 2020-01-29 | | 阅读数

Introduction

Python is the first computer programming language I learnt so that I have some thoughts to share about my experience. Compared with human mind, in some ways computer works the same. Most of the programs I learnt so far work based on two logic: True or False and if or else. Computer will make choice according to the content we type in just like human. However, I want to talk more about differences between computer language and human language. When we give somebody an order, we focus on the thesis, details are not so important. Nevertheless, it is totally different in computer language. Every detail must be correct in the code we type, such as the indented block, the symbol, the format of your code and even a blank. If there is only one mistake, the whole program will not work. On the other hand, symbols in computer language are have different meaning compared with it in math. For example, when we want two Boolean values equal, we need to type ‘==’ instead of ‘=’. As a beginner, I have a lot of trouble to use correct symbols when I write code.

Impression

I used to have a question after a few weeks’ learning: Computer is used to help us work more convenient and quickly, but why I seem to spend more time on writing a program? I can finish it much more quickly by myself. The problem bothered me until the teacher gave us an example: A teacher wants to know 10 students average mark after he puts10 results in. I just spent 3 minutes to finish the code instead of adding 10 marks and dividing 10. In my opinion, computer is just a like a little child which only can do any complex calculation and never forget things. In order to let him to solve difficult questions, I need to teach him how to do it and I only teach him once. After that, I just need to type my data and he will give me the answer. I think that is the reason we learn the different computer language.

A flowchart is an important step in writing a program. Flowcharts show the logic you need to write a program in a simple way, helping us tease out the key steps. There are only two sections of the flow chart: the execution section and the judgment section.

Examples

Now I am going to introduce some program that I write.

First program is help customers order their food. During rush hours, many artificial Windows may become. If shop manager can add the self-service ordering machine in the store can greatly reduce the labor workload, in McDonald’s now has such equipment.

Second one is to calculate the monthly payment in a bank. It is extremely difficult for beginners like me to calculate all kinds of figures. Not only the meaning of some data is professional, but also the complex calculation annoy me a lot. That’s the reason that I strongly suggest to use this kind of program.

Here is a flowchart I drew while writing the program. This is a simple login procedure just like QQ. When you put correct name and password in, it will let you pass. We can see that only a simple procedure needs a amount of code and logic, so it is necessary to draw a flowchart.
